## Usage

``` bash
 echo 'x = 0xDEAD
   y = 0xBEEF
   x | y' | ./ceval            # → 0xDEAD / 0xBEEF / 0xFEEF
```

``` bash
   ./ceval -b                  # binary output instead of hex
```

Bit-width inference (the zero-padding rule):

| Assignment        | Inferred width       | Example output |
|:------------------|:---------------------|:---------------|
| x = 0xFF          | 8-bit (2 hex digits) | 0xFF           |
| x = 0x1234        | 16-bit (4 digits)    | 0x1234         |
| x = 0xDEADBEEF    | 32-bit (8 digits)    | 0xDEADBEEF     |
| x = 255 (decimal) | 32-bit (C int)       | 0x000000FF     |
| int x = …         | 32-bit (declared)    | 8 hex digits   |
| uint8_t x = …     | 8-bit (declared)     | 2 hex digits   |


Output is padded to the widest variable referenced in the expression (so small_8bit + big_32bit pads to 32 bits). Reassigning an existing variable preserves its type; use an explicit int x = … to change it.

Supports: & | ^ ~ << >>, + - * / %, && || !, == != < > <= >=, ++x x++, compound += |= &= etc., (int) casts (dropped), sizeof(type), ternary a ? b : c, // comments, trailing ;.
